# Lab 06: Docker Bench Security Audit

## Выявленные уязвимости

### 1. Привилегированные контейнеры
- **Порт**: N/A
- **Сервис**: web, vulnerable-web
- **Уровень критичности**: HIGH
- **Причина**: Контейнеры запущены с флагом `privileged: true`
- **Сценарий атаки**: Злоумышленник при компрометации контейнера получает root-доступ к хосту

### 2. Host network mode
- **Порт**: N/A
- **Сервис**: vulnerable-web
- **Уровень критичности**: CRITICAL
- **Причина**: Используется `network_mode: host`
- **Сценарий атаки**: Контейнер имеет полный доступ к сетевому стеку хоста

### 3. Секреты в переменных окружения
- **Порт**: N/A
- **Сервисы**: web, app, postgres, vulnerable-web
- **Уровень критичности**: HIGH
- **Причина**: Пароли и ключи хранятся в ENV
- **Сценарий атаки**: Утечка credentials через `docker inspect` или компрометацию контейнера

### 4. Монтирование docker.sock
- **Порт**: N/A
- **Сервис**: vulnerable-web
- **Уровень критичности**: CRITICAL
- **Причина**: `-v /var/run/docker.sock:/var/run/docker.sock`
- **Сценарий атаки**: Полный контроль над Docker daemon хоста

### 5. Использование :latest тегов
- **Порт**: N/A
- **Сервисы**: web, postgres, vulnerable-web, debug-shell
- **Уровень критичности**: MEDIUM
- **Причина**: Образы используют `:latest` тег
- **Сценарий атаки**: Непредсказуемые обновления, возможные уязвимые версии

### 6. Запуск от root
- **Порт**: N/A
- **Сервисы**: Все контейнеры
- **Уровень критичности**: HIGH
- **Причина**: Отсутствует инструкция USER
- **Сценарий атаки**: Привилегии root внутри контейнера при побеге

# CIS Benchmark vs Trivy

## CIS Benchmark (Docker Bench Security)
- **Что проверяет**: Конфигурацию хоста, Docker daemon, права доступа
- **Тип проверок**: Настройки безопасности, привилегии, монтирования
- **Результат**: Да/Нет соответствие CIS рекомендациям
- **Когда нужен**: При развёртывании Docker хоста, hardening

## Trivy
- **Что проверяет**: CVE уязвимости в пакетах и зависимостях
- **Тип проверок**: Версии пакетов, известные уязвимости
- **Результат**: Список CVE с уровнями критичности
- **Когда нужен**: В CI/CD, перед деплоем, регулярное сканирование

## Комбинация
Оба инструмента необходимы для комплексной безопасности контейнеров.
