## 1. Описание обнаруженных недостатков архитектуры и рисков

### Выявленная проблема
Административная консоль веб-сайта компании доступна неограниченному кругу лиц в сети Интернет. Консоль содержит:
- Записи с запросами о приобретении продукции с персональными данными клиентов
- Коммерческие предложения (объёмы, цены, специальные условия)
- Логи с IP-адресами администраторов

### Категории данных
- Персональные данные (ПДн) граждан РФ и ЕС (ФИО, контактная информация, адреса доставки)
- Коммерческая тайна (ценообразование, специальные условия, объёмы закупок)
- Инсайдерская информация (логи администраторов, паттерны доступа)

### Идентифицированные риски

| № | Риск | Вероятность | Ущерб | Уровень |
|---|------|-------------|-------|---------|
| R1 | Несанкционированный доступ к ПДн | Высокая | Критический | HIGH |
| R2 | Утечка коммерческой информации | Средняя | Высокий | HIGH |
| R3 | Компрометация учётных данных администраторов | Средняя | Критический | CRITICAL |
| R4 | Нарушение требований GDPR и 152-ФЗ | Высокая | Критический | CRITICAL |
| R5 | Репутационные потери | Средняя | Высокий | HIGH |

## 2. Меры снижения рисков (митигация)

### R1: Несанкционированный доступ к ПДн

| Параметр | Значение |
|----------|----------|
| **Проблема** | Админская консоль доступна из публичного интернета |
| **Решение** | Ограничение доступа по сети и внедрение аутентификации |
| **Ценность** | Защита ПДн, выполнение требований GDPR/152-ФЗ |
| **Приоритет** | CRITICAL |

**Технические меры:**
1. Закрыть доступ к админской консоли на сетевом уровне (allowlist IP-адресов сотрудников)
2. Перенести консоль во внутреннюю сеть/VPN
3. Внедрить многофакторную аутентификацию (MFA)
4. Настроить автоматическое логирование всех действий

**Эффективность:** 95% снижение риска

---

### R2: Утечка коммерческой информации

| Параметр | Значение |
|----------|----------|
| **Проблема** | Коммерческие предложения доступны без авторизации |
| **Решение** | Шифрование данных и разграничение доступа |
| **Ценность** | Сохранение конкурентных преимуществ |
| **Приоритет** | HIGH |

**Технические меры:**
1. Шифрование коммерческих данных при хранении и передаче
2. Внедрение RBAC (ролевой модели доступа)
3. Маскирование чувствительных полей в интерфейсе
4. Настройка DLP-системы

**Эффективность:** 85% снижение риска

---

### R3: Компрометация учётных данных администраторов

| Параметр | Значение |
|----------|----------|
| **Проблема** | Логи с IP-адресами доступны злоумышленникам |
| **Решение** | Защита логов и усиление аутентификации |
| **Ценность** | Предотвращение захвата контроля над системой |
| **Приоритет** | CRITICAL |

**Технические меры:**
1. Регулярная ротация паролей и ключей доступа
2. Внедрение SIEM для мониторинга подозрительной активности
3. Обучение персонала правилам безопасности
4. Использование менеджера паролей

**Эффективность:** 90% снижение риска

---

### R4: Нарушение требований GDPR и 152-ФЗ

| Параметр | Значение |
|----------|----------|
| **Проблема** | Несанкционированный доступ к ПДн граждан ЕС и РФ |
| **Решение** | Приведение инфраструктуры к требованиям регуляторов |
| **Ценность** | Избежание штрафов и юридических последствий |
| **Приоритет** | CRITICAL |

**Требования по GDPR (ст. 32):**
- Псевдонимизация и шифрование ПДн
- Обеспечение конфиденциальности, целостности и доступности
- Регулярное тестирование мер безопасности

**Требования по 152-ФЗ (ст. 19):**
- Назначение ответственного за обработку ПДн
- Применение сертифицированных средств защиты
- Уведомление Роскомнадзора об инцидентах

**Эффективность:** 100% соответствие при выполнении мер

---

### R5: Репутационные потери

| Параметр | Значение |
|----------|----------|
| **Проблема** | Утечка данных подрывает доверие клиентов |
| **Решение** | Прозрачная политика безопасности и быстрый response |
| **Ценность** | Сохранение лояльности клиентов и бизнеса |
| **Приоритет** | HIGH |

**Технические меры:**
1. Разработка плана реагирования на инциденты (IRP)
2. Внедрение системы уведомлений об утечках (72 часа по GDPR)
3. Регулярный внешний аудит безопасности
4. Публикация политики конфиденциальности

**Эффективность:** 70% снижение репутационного ущерба

## 4. Заключение

### Итоговая оценка
Текущий уровень риска до внедрения мер: **CRITICAL**

Остаточный уровень риска после внедрения мер: **LOW**

### Приоритетные действия (0-30 дней)
1. Немедленно закрыть доступ к админской консоли из внешней сети
2. Внедрить MFA для всех административных аккаунтов
3. Провести инвентаризацию всех ПДн в системе

### План долгосрочных мероприятий (30-90 дней)
1. Внедрение SIEM и DLP систем
2. Разработка и утверждение IRP
3. Проведение обучения персонала
4. Сертификация по ISO 27001

## 5. Compliance-соответствие

| Регулятор | Статус | Меры для приведения |
|-----------|--------|---------------------|
| GDPR | Не соответствует | Шифрование, уведомления об утечках |
| 152-ФЗ | Не соответствует | Локализация данных, сертифицированные СЗИ |
| ISO 27001 | Не соответствует | Внедрение СУИБ |
| NIST SP 800-30 | Частично | Требуется полная оценка рисков |
